# myRecipe

配方管理系统的前端原型项目。

当前这一版重点不是继续堆功能，而是先把项目底层结构整理清楚，让后续新增模块时不再回到单文件开发模式。

## 当前结构

- `index.html`
  页面入口，只负责挂载资源和页面骨架。

- `styles/`
  样式分层目录。
  `tokens.css` 放设计变量。
  `base.css` 放基础样式。
  `layout.css` 放布局和响应式。
  `components.css` 放组件样式。
  `main.css` 作为统一入口。

- `src/data/`
  演示数据和初始化数据。

- `src/state/`
  全局运行时状态。

- `src/core/`
  DOM 引用等基础设施代码。

- `src/features/`
  业务功能模块，按导航和功能区拆分。

- `src/utils/`
  工具函数和格式化方法。

- `docs/`
  项目结构和后续演进文档。

## 已完成的底层铺垫

- 侧边栏支持展开和收起。
- 入口脚本改为模块化加载。
- 业务逻辑从单一 `index.js` 拆分到多个功能模块。
- 样式从单一 `index.css` 拆分成多层结构。
- 新增结构文档，方便后续扩展。

## 后续建议

下一阶段适合继续做这些基础能力：

1. 增加 `src/services/`，开始抽 API、缓存和同步逻辑。
2. 增加 `src/constants/`，统一状态枚举和配置。
3. 增加 `tests/`，先覆盖纯函数和业务计算。
4. 等功能复杂度上来后，再迁移到 Vite 或更完整的工程体系。
